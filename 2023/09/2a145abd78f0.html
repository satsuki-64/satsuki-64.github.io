1.1 什么是解决 computational 的问题

什么是解决 computational 的问题
设置一系列的输入，产生一系列的输出；

  • Binary relation from problem inputs to correct outputs
  • 在这节课上，重点不是输入与答案之间的联系，而是设置一定的 predicate，然后制作算法，使其可以适应各样的 input 以及 output space；

什么是这个课程当中指出的算法
目标在更加普遍的问题，他们有更加灵活的尺寸的 input，在不同尺寸的情况下都可以解决它们；

  • 算法是某种函数：
    • f : I -> O

如何证明算法的正确性
算法可能会有不固定大小的 Input 以及 Output，此时如何证明其算法作为函数的正确性？

  • Induction：归纳法
    • 写下恒定大小的 piece of code，可以接受任意大小的数据，然后通过 Loop 来计算这些不固定大小的数据；

1.2 算法的正确性

Programs/algorithms have fixed size, so how to prove correct?

对于较小数量的问题，可以使用穷举法进行解除；而对于 arbitrarily large inputs, algorithm must be recursive or loop in some way；

引出的问题
如何通过对于问题的描述 (比如函数输出如此这般的结果的时候为正确，这种正确如何证明)，普遍化一个正确性并且归纳法证明；

Example：Proof of correctness of birthday matching algorithm

  • 如果
  • Induct on k: the number of students in record
  • Hypothesis:
    • if first k contain match, returns match before interviewing student k + 1
    • 如果学生 k 被包含并 match，返回这之后学生数量的 k+1；
  • Base case: k = 0, first k contains no match
  • Assume for induction hypothesis holds for k = k 0 , and consider k = k 0 + 1
    • If first k 0 contains a match,
      • already returned a match by induction
    • Else first k 0 do not have match,
      • so if first k 0 + 1 has match, match contains k 0 + 1
  • Then algorithm checks directly whether birthday of student k 0 + 1 exists in first k0

1.3 算法的效率

什么是效率
Efficiency：不只是速度的快慢，还有和其他算法的比较；

  • 因为在不同的设备、不同的数据情况下，速度很难进行绝对的比较；
  • 分析方法：不涉及时间，但是使用 Count Fundamental Operation；
    • asymptotic analysis
    • 渐进分析
  • 符号系统
    • asymptotic notation
    • Input 的 size：n
    • Upper bounds (O)
    • lower bounds (Ω)
    • tight bounds (Θ) ∈, =, is, order
  • 时间复杂度
    • ologn 几乎类似于常数（随着数字变大）；

1.4 计算模型

Model Of Conputation
对于算法所做出的一些假设

  • Specification for what operations on the machine can be performed in O (1) time
  • Model in this class is called the Word-RAM
    • Machine word: block of w bits (w is word size of a w-bit Word-RAM)
      • 对于字（WORD）大小的定义；
    • 使用 RAM 进行数据的存储与访问：支持随机读写；
