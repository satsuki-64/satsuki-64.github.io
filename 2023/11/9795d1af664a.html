Part 2：Tmux
简单流程：

  1. 新建会话 tmux new -s my_session。
  2. 在 Tmux 窗口运行所需的程序。
  3. 按下快捷键 Ctrl+b d 将会话分离。
  4. 下次使用时，重新连接到会话 tmux attach-session -t my_session

指令：

  • 退出 Tmux
    • exit
    • Ctrl+D
  • 前缀键
    • Ctrl+b
  • 新建会话
    • 创建制定名称的会话
      • tmux new -s <session-name>
  • 分离会话
    • 快捷键
      • 前缀键+d
    • 命令
      • tmux detach
  • 查看现有会话
    • tmux ls
  • 接入会话
    • tmux attach -t <会话编号>
    • tmux attach -t <会话名称>
  • 杀死会话
    • tmux kill-session -t <会话名称|编号>
  • 切换会话
    • tmux switch -t <会话名称|编号>
  • 窗口划分
    • 划分上下两个
      • tmux split-window
    • 划分左右两个
      • tmux split-window -h
  • 移动光标
    • tmux select-pane -UDLR
    • U：上方
    • D：下方
    • L：左方
    • R：右方
    • 快捷键
    • 快捷键

Part 3：Linux 常用 API

  • 文件系统：
    • 普通 IO
      • Read
      • Open
      • Lseek
      • Close
    • 标准 IO
      • 缓冲类型
        • 全缓冲
        • 行缓冲
        • 不带缓冲
      • Fopen|fdopen|freopen
      • Fclose
      • Fread
  • 进程控制开发
    • 进程启动
      • 调度启动
      • 手动启动
        • 前台启动
        • 后台启动
    • 进程控制基础
      • Fork ()
      • Exec 系列
        • Exec+
        • P：完整 path
        • L：一行参数
        • V：完整参数
        • E：完整环境
      • 终止
        • Exit
      • Wait
        • Waitpid 是非阻塞版本
    • 守护进程
      • 生存期较长的进程，通常独立于控制终端并且周期性地执行某种任务或等待处理某些发生的事件。如果想让某个进程不因为用户、终端或者其他的变化而受到影响，那么就必须把这个进程变成一个守护进程。
    • 守护进程创建
      • 创建子进程，父进程推出
      • 在子进程中创建新的会话
        • Setsid：创建新会话
      • 改变当前目录为根目录
        • Chdir：改变目录
      • 重设文件权限掩码
        • Umask
      • 关闭文件描述符
        • Close
    • 出错处理
      • Syslog：系统日志
      • Openlog
  • 进程间通信
    • 管道
      • 描述
        • 是基于文件描述符的通信方式，当一个管道建立时，它会创建两个文件描述符 fds[0]和 fds[1]，其中 fds[0]固定用于读管道，而 fd[1]固定用于写管道。
      • API
        • Pipe ()接受 fd
        • Fd[2]：管道的两个文件描述符，之后就可以直接操作这两个文件描述符
      • 标准流管道
        • 只有一个 popen、pclose：表示标准 pipe
        • 创建一个管道。
        • Fork ()一个子进程。
        • 在父子进程中关闭不需要的文件描述符。
        • 执行 exec 函数族调用。
        • 执行函数中所指定的命令
      • FIFO：有名管道
        • 概念
          • 无名管道，它只能用于具有亲缘关系的进程之间。有名管道的出现突破了这种限制，它可以使互不相关的两个进程实现彼此通信。
          • 通过路径名来指出
        • API
          • 创建：mkfifo（）
            • 在创建管道成功之后，就可以使用 open ()、read ()和 write ()这些函数
          • O_RDONLY：读管道
          • O_WRONLY：写管道
          • O_RDWR：读写管道
          • O_NONBLOCK：非阻塞_
      • 信号
        • 各种信号类型
        • 发送捕捉
          • Fill
          • Raise
          • Alarm
          • Pause
        • 信号处理
          • Signal | sigaction
      • 信号集函数组
        • Sigemptyset ()：将信号集合初始化为空。
        • Sigfillset ()：将信号集合初始化为包含所有已定义的信号的集合。
        • Sigaddset ()：将指定信号加入到信号集合中去。
        • Sigdelset ()：将指定信号从信号集合中删除。
        • Sigismember ()：查询指定信号是否在信号集合之中。
        • 定义信号集合
        • 设置信号屏蔽
        • 定义信号处理函数

Part 5：CMake 使用

基础语法特性

  • 基础语法格式
    • 指令格式
      • 指令(参数1 参数2 ...)
      • 参数之间使用空格或分号隔开
    • 变量
      • 创建变量
        • set(变量名 变量值)
      • 使用变量：使用 ${} 的方式从变量名取值，在 IF 控制语句中可以直接使用变量名；

重要指令

  • 指定 CMake 最小版本要求
    • cmake_minimum_required(VERSION versionNumber [FATAL_ERROR])
  • 定义工程名称
    • project(projectname [CXX][C][Java])
    • 例：project(HELLOWORLD)
  • 定义变量
    • 语法：set(VAR [VALUE][CACHE TYPE DOCSTRING[FORCE]])
    • 例：set(SRC test1.cpp test2.cpp)
  • 目录定义变量
    • 语法：aux_source_directory(dir VARIABLE)
  • 添加多个特定头文件搜索路径
    • 语法：include_directories([AFTER|BEFORE][SYSTEM] dir1 dir2 ...)
    • 例：include_directories(/usr/include/myinlcudefloder ./include)
  • 添加库文件搜索路径
    • 语法：link_directories(dir1 dir2 ...)
  • 生成库文件
    • 语法： add_library(libname [SHARED|STATIC|MODULE][EXCLUDE_FROM_ALL] source1 source2 ...)
    • 例：add_library(NewLib STATIC ${SRC})
  • 添加编译参数
    • 语法：add_complier_options(<option> ...)
    • 例：add_complier_options(-Wall -O2)
  • 生成可执行文件
    • 语法：add_executable(exename source1 source2 ...)
    • 例：add_executable(main main.cpp)
  • 为 target 添加需要链接的动态链接库
    • 语法：target_link_libraries(target library1 library2 ...)
  • 向当前工程添加存放源文件的子目录，并可以制定二进制文件存放的位置
    • 语法：add_subdirectory(source_dir [binary_dir][EXCLUDE_FROM_ALL])

常用变量

  • 设定编译类型
    • 语法： set(CMAKE_BUILD_TYPE Debug|Release)
  • 可执行文件输出路径
    • 变量：EXECUTALBE_OUTPUT_PATH
  • 库文件输出路径
    • 变量：LIBRARY_OUTPUT_PATH
